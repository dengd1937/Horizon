## [Codex/Claude Code 上下文压缩与提示缓存改进](https://x.com/dotey/status/2070904833939329477) ⭐️ 6.0/10

一位开发者报告称，Codex 和 Claude Code 的上下文压缩与提示缓存已显著改进，降低了长时间会话的成本，并强调了用于保持上下文纯净的 fork 功能。 这些改进使得维持长时间编码会话更加实用，无需承担过高的 token 成本，从而提升了使用 AI 编码助手的开发者的生产力。 上下文压缩会在接近上下文窗口限制时自动总结旧消息，而提示缓存通过重用缓存的响应来降低延迟和 API 成本。fork 功能允许从特定点分支，仅保留之前的历史记录以保持上下文纯净。

twitter · dotey · 6月27日 16:19

**背景**: 大型语言模型具有有限的上下文窗口，长对话可能超出限制，导致错误或高成本。上下文压缩通过总结旧消息来适应窗口，而提示缓存存储常用提示以避免重复计算。fork 功能创建一个包含部分历史记录的新会话，用于在不丢失原始上下文的情况下探索替代路径。
