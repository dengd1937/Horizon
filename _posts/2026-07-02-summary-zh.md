## [Claude Code 被指偷偷给中国代理用户打水印](https://x.com/dotey/status/2071995057247297924) ⭐️ 9.0/10

安全研究员 Adnane Khan 在 GitHub 上发布逆向分析报告，指出 Anthropic 的 Claude Code v2.1.193 至 v2.1.196 版本在系统提示词中嵌入 Unicode 差异，用于检测中国代理用户。该工具会检查代理域名是否在包含 147 个中国域名的列表中，并检查时区，然后通过修改日期分隔符或撇号字符来编码检测结果。 这一发现动摇了用户对需要深度系统权限的 AI 编程工具的信任，因为水印机制是隐蔽且未公开的。它引发了严重的隐私和透明度担忧，尤其是 Claude Code 能够读取代码仓库、运行命令和修改文件。 该机制使用隐蔽信道：当时区为 Asia/Shanghai 或 Asia/Urumqi 时，日期分隔符从 '-' 改为 '/'；并使用四种不同的 Unicode 撇号字符来指示代理域名是否匹配列表、AI 实验室关键词、两者都匹配或都不匹配。域名列表经过 XOR-91 编码，包含百度、阿里巴巴、字节跳动、Moonshot AI、MiniMax、阶跃星辰等中国大公司和 AI 实验室。水印是代理触发的，仅在用户设置 ANTHROPIC_BASE_URL 环境变量指向非官方端点时激活。

twitter · dotey · 6月30日 16:31 · [原文链接](https://x.com/dotey/status/2071995057247297924)

**背景**: Claude Code 是 Anthropic 推出的 AI 编程代理，能够读取代码仓库、执行终端命令和修改文件。系统提示词是在每次请求开始时发送给 AI 模型的指令，用于设置上下文。Unicode 同形字是视觉上相同但码点不同的字符，常被用于欺骗或隐写术。ANTHROPIC_BASE_URL 环境变量允许用户通过代理路由 API 请求，这在企业网关或受限网络环境中很常见。

**社区讨论**: Reddit 和 GitHub 上的讨论普遍谴责这种做法是信任的背叛，许多用户称其类似间谍软件。一些评论者指出，虽然该机制作为反滥用手段很弱，但它不公平地针对了合法用户，并破坏了透明度。

## [何时审阅 AI 生成的代码：实用指南](https://x.com/dotey/status/2072482750636966031) ⭐️ 8.0/10

@laike9m_ 发布并被 @dotey 转推的一条热门推文，为智能体编程时代的人工审阅 AI 生成代码提供了细致的框架。它主张自动化测试和 AI 代码审查可以处理 90%的检查工作，人类只需专注于架构设计。该推文还建议利用 AI 生成提交摘要和架构图，以简化人工审阅流程。 随着 AI 编程助手越来越强大，开发者面临大量 AI 生成的代码，传统的逐行审查变得不切实际。该框架提供了一种可扩展的方法，平衡了效率与质量，帮助团队在不使人工审查者不堪重负的情况下保持代码完整性。它还强调了人类在架构方面的持久价值，这是 AI 目前无法复制的。 该推文建议所有可自动验证的行为都应通过自动化测试覆盖，AI 可以轻松编写单元测试，但复杂的集成测试仍需人工搭建。AI 代码审查可以处理 90%的检查（语法、注释、边界条件、日志记录），前提是团队有严格的代码规范。剩下的 10%需要人类判断的是架构设计，这通常与项目相关且难以标准化。为了提高审查效率，人类应阅读 AI 生成的提交摘要和架构图，而不是完整代码。

twitter · dotey · 7月2日 00:49 · [原文链接](https://x.com/dotey/status/2072482750636966031)

**背景**: 智能体编程（Agentic coding）是指使用自主 AI 代理，在最少人工干预下规划、编写、测试和修改代码，其操作在项目级别而非文件级别。AI 代码审查工具如 CodeRabbit 和 Graphite 可以自动检查代码风格、检测错误并提出改进建议，但它们缺乏对业务逻辑和项目特定架构的理解。该推文基于一个日益被认可的观点：虽然 AI 擅长生成语法正确的代码，但常常做出糟糕的架构决策，因此对于长期维护的项目，人工监督至关重要。

## [Anthropic 推出面向科研人员的 Claude Science AI 工作台](https://x.com/dotey/status/2072352618676953539) ⭐️ 8.0/10

Anthropic 发布了 Claude Science，一个专为科学研究人员设计的 AI 工作台。它集成了 60 多个科学数据库和多个 AI 代理，将计算生物学工作流程整合到单一环境中，支持通过自然语言驱动。该产品现已在 macOS 和 Linux 上公开测试，面向 Pro、Max、Team 和 Enterprise 订阅用户开放。 Claude Science 标志着 Anthropic 的战略转变——从单纯出售模型能力转向拥有特定行业的操作层，类似于 Claude Code 在软件开发领域的作用。通过整合分散的研究工具和数据库，它可能极大加速科学发现并提高可重复性。此举使 Anthropic 能够在生命科学 AI 市场与 OpenAI 的 GPT-Rosalind 直接竞争。 Claude Science 使用现有的 Claude 模型（包括 Opus 4.8），没有专门训练生物学能力；它专注于工作流程集成而非模型改进。平台有一个主 AI 代理，将任务分派给专业子代理，还有一个审查代理用于验证引用和结果，并支持通过 SSH 连接到本地高性能计算集群进行本地运算。每张生成的图表都附带完整代码、运行环境和对话历史，以确保可重复性。Anthropic 将资助最多 50 个研究项目，每个项目最高 3 万美元，外加最多 2000 美元的 Modal 计算资源。
