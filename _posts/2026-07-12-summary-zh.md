## [GPT-5.6 Sol Ultra 解决 50 年数学猜想](https://x.com/OpenAI/status/2075644390689898690) ⭐️ 10.0/10

OpenAI 宣布其 GPT-5.6 Sol Ultra 模型生成了循环双覆盖猜想的证明，这是一个图论中 50 年未解的难题。该证明在不到一小时内由 64 个子代理并行工作完成。 这标志着 AI 系统首次自主解决了一个长期存在的数学猜想，展示了先进的推理和协作能力。它可能通过提供生成证明的新工具，加速数学及其他科学领域的研究。 循环双覆盖猜想断言每个无桥图都有一个循环集合，使得每条边恰好被覆盖两次。GPT-5.6 Sol Ultra 使用了 64 个子代理，每个处理问题的一部分，并在不到一小时内生成了证明。OpenAI 公开分享了提示词和生成的证明。

twitter · OpenAI · 7月10日 18:12 · [原文链接](https://x.com/OpenAI/status/2075644390689898690)

**背景**: 循环双覆盖猜想由 Tutte、Itai、Rodeh、Szekeres 和 Seymour 等数学家提出，是图论中的核心问题，与图嵌入和四色定理相关。GPT-5.6 Sol Ultra 是 OpenAI 的最新模型，采用子代理架构，支持并行解决问题。

**社区讨论**: X（原 Twitter）上的早期反应对速度和异常简短的证明印象深刻。一些研究人员表示谨慎乐观，指出该证明仍需同行评审。

**标签**: `#AI`, `#OpenAI`, `#GPT-5.6`, `#mathematics`, `#breakthrough`

## [OpenAI 发布 GPT-5.6，成本降低 25 倍](https://x.com/OpenAI/status/2075686461693898868) ⭐️ 9.0/10

OpenAI 宣布推出 GPT-5.6 系列模型，包括 Sol、Terra 和 Luna。其中 Luna 版本在最高推理设置下性能超越 GPT-5.5，而成本仅为后者的 1/25。 这一突破使先进 AI 在全球范围内更易获取，特别是在健康智能应用方面。大幅降低成本可能推动 AI 驱动的医疗保健民主化，使资源受限的环境也能广泛采用。 GPT-5.6 Luna 是最快且最具成本效益的选项，适合快速编辑和迭代。该系列还包括面向开发者和企业的 Sol，以及性能均衡的 Terra。早期社区测试表明，Luna 在最大努力模式下可能会走捷径，但总体编码性能强劲。

twitter · OpenAI · 7月10日 20:59 · [原文链接](https://x.com/OpenAI/status/2075686461693898868)

**背景**: GPT-5.5 是 OpenAI 之前的旗舰模型，提供高推理能力但计算成本高昂。新的 GPT-5.6 系列引入多个层级以满足不同用例和预算。健康智能指分析医疗数据以提供洞察、诊断或个性化建议的 AI 系统。

**社区讨论**: Reddit 讨论指出，Luna 在许多用例中表现出色，但在最大努力模式下可能会走捷径，暗示速度和彻底性之间存在权衡。一些用户对成本降低印象深刻，而另一些则质疑该模型在复杂任务中的可靠性。

**标签**: `#OpenAI`, `#GPT-5.6`, `#AI`, `#health intelligence`, `#cost reduction`

OpenAI 已向全球所有 ChatGPT 用户全面推出 GPT-Live。此外，公司还在整个周末将所有人的语音使用额度翻倍，以鼓励更多测试。 这标志着 ChatGPT 语音模式的重大升级，实现了更自然、更响应的人机交互。使用额度的增加让用户能更自由地探索新功能，可能推动更广泛的采用。 GPT-Live 是新一代语音模型，专为自然的人机交互设计，现已为 ChatGPT Voice 提供支持。此次推出是全球性的、立即生效，用户无需额外付费。翻倍的语音使用额度仅限周末，之后恢复正常限制。

**背景**: OpenAI 主要以 GPT-4 等 AI 软件闻名，但近年来开始向硬件领域扩张。2025 年 5 月，它收购了由 Jony Ive 和前苹果高管创立的 io Products 公司，旨在开发 AI 硬件。苹果长期以来一直严格保护其商业机密，尤其是在产品设计和工程方面。诉讼指控前苹果员工将机密信息带到 OpenAI，从而帮助其开发竞争性硬件。

**背景**: 通用越狱是一种能够绕过多个 AI 模型或输入的安全护栏的技术，构成重大安全风险。AI 中的生物安全挑战侧重于防止模型提供可能促成生物威胁创建的信息。OpenAI 的生物漏洞赏金计划是更广泛的行业努力的一部分，旨在对 AI 系统进行红队测试以发现安全漏洞。

## [ChatGPT、Work 和 Codex 的区别与额度说明](https://x.com/dotey/status/2075652538058109385) ⭐️ 7.0/10

一份详细的问答澄清了 ChatGPT Chat、Work 和 Codex 之间的区别，说明 Chat 回答问题，Work 作为智能体处理商务任务，Codex 专注于代码仓库。同时揭示 Chat 和 Work/Codex 的额度独立，Work 和 Codex 共享一个单独的“智能体用量”池。 这一澄清帮助用户和开发者更有效地使用 OpenAI 生态系统，避免对工具功能和额度管理的困惑。理解这些差异可以优化使用和支出，特别是对于同时依赖 Work 和 Codex 的团队。 Work 通过插件连接 Slack、Gmail、Google Drive 等业务应用，生成文档和幻灯片等交付物；Codex 读取代码仓库，执行修复 bug、提交 PR 等任务。Work 支持定时任务，可设置一次性、重复或事件触发，但任务每小时不能运行超过一次。Codex 和 Work 使用相同的底层智能体技术，但针对不同场景定制。

twitter · dotey · 7月10日 18:45 · [原文链接](https://x.com/dotey/status/2075652538058109385)

**背景**: OpenAI 提供多种 AI 工具：ChatGPT Chat 用于对话问答，Work 用于自动化商务工作流，Codex 用于编程辅助。这些工具是 OpenAI 将 AI 集成到各专业领域的一部分。Chat 和智能体工具（Work/Codex）的额度独立，意味着大量使用一个不会影响另一个，但 Work 和 Codex 共享同一个智能体用量池。

## [UI 与代码：棉花比喻](https://x.com/dotey/status/2076054180251218197) ⭐️ 6.0/10

@dotey 的一条推文将 UI 比作衣服的外观，将代码比作棉花的材质，暗示人们更关心表面的美观而非内在的质量。推文引用了之前关于 AI 生成的 UI 被嫌弃而 AI 生成的代码却被欣然接受的评论。 这个比喻突显了软件工程中常见的偏见：视觉上的精致往往掩盖了代码质量，影响产品的评价。它引发了关于在开发的不同方面接受 AI 生成内容时存在双重标准的讨论。 推文使用服装的比喻：人们关心衣服好不好看、是否撞衫，但很少在意棉花的质量。这被应用于 UI（面子）和代码（里子），暗示用户和开发者更重视 UI 美观而非代码质量。

twitter · dotey · 7月11日 21:21 · [原文链接](https://x.com/dotey/status/2076054180251218197)

**背景**: 在软件开发中，UI（用户界面）是应用程序的可见部分，而代码是底层逻辑。这个比喻反映了前端和后端优先级之间的常见张力。引用的关于 AI 生成内容的评论指出了在设计和编码中对 AI 接受程度的不同。

## [用户批评 Claude Code 桌面版右侧面板设计](https://x.com/dotey/status/2075729483026305070) ⭐️ 5.0/10

一位用户在 X 上批评 Claude Code 桌面应用的右侧面板设计，称其经常将浏览器视图缩到很小。此外，任务完成后生成的结果不可点击，用户必须手动复制文件名进行搜索。 这一反馈突显了旨在简化编码流程的开发者工具中一个重要的可用性问题。如果不解决，可能会影响用户采用率和满意度，尤其是与提供可点击预览的 Codex 等竞品相比。 用户将这种体验与 Codex 进行了不利比较，后者生成的结果可点击，并能预览或在 VSCode 中打开。Claude Code 桌面应用包含一个沙盒化且可配置的应用内浏览器，但右侧面板设计据称遮挡了主浏览器视图。

twitter · dotey · 7月10日 23:50 · [原文链接](https://x.com/dotey/status/2075729483026305070)

**背景**: Claude Code 是 Anthropic 推出的 AI 编程代理，运行在桌面应用中，提供管理多个会话的图形界面、集成终端、文件编辑器和可视化差异审查。Codex 是 OpenAI 的竞品 AI 编程工具，提供带有内置工作树和云环境的代理编码指挥中心。用户的批评聚焦于 Claude Code 桌面应用的 UI/UX，特别是承载应用内浏览器的右侧面板。

**标签**: `#Claude Code`, `#UI/UX`, `#Developer Tools`, `#Feedback`
